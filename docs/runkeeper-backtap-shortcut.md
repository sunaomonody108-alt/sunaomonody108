# Runkeeper 背面タップ ショートカット 設定ガイド

iPhoneの「背面タップ」機能を使って、Runkeeperのランニングを1タップで開始・停止するショートカットです。

---

## 完成イメージ

| タップ | 動作 |
|--------|------|
| 背面を2回タップ（初回） | Runkeeperを開いてランニング開始 |
| 背面を2回タップ（再度） | Runkeeperを開いてランニング停止 |

> **ポイント:** 同じ「背面タップ」の動作で「開始→停止→開始…」とトグル切り替えします。現在の状態はiCloudのファイルで管理します。

---

## 事前準備

### 1. RunkeeperのSiriショートカットを設定する

Runkeeperアプリ内でSiriショートカットを登録しておくと、ショートカットから直接起動できます。

1. **Runkeeper** を開く
2. 画面左上の **ユーザーアイコン** → **設定** → **Siriショートカット**（またはiOSの「設定」→「Siri と検索」→「Runkeeper」）
3. 「ランニングを開始」のショートカットを追加

> Siriショートカットが見当たらない場合は、一度Runkeeperでランニングを記録するとドネーションされます。

### 2. ショートカットアプリでフォルダを確認

「ショートカット」アプリ → 「iCloud Drive」への書き込みが許可されているか確認します（初回実行時に自動で許可を求められます）。

---

## ショートカットの作成手順

### ショートカットアプリを開く

1. iPhoneで **ショートカット** アプリを起動
2. 右上の **「+」** をタップ → 新規ショートカット
3. 右上の **「…」** → 名前を **「Runkeeper トグル」** に変更

---

### アクションを追加する（順番通りに）

#### ① 現在の状態を読み取る

```
アクション: 「ファイルを取得」
  - 場所: iCloud Drive
  - ファイルパス: Shortcuts/runkeeper_state.txt
  - エラー時は続行: オン（初回はファイルがないため）
```

#### ② 取得した状態を変数に保存

```
アクション: 「変数を設定」
  - 変数名: currentState
  - 値: ファイルを取得の結果
```

#### ③ 条件分岐（if文）

```
アクション: 「if」
  - 入力: currentState
  - 条件: 次と等しい
  - 値: running
```

**「running の場合」（ランニング中 → 停止）:**

```
アクション④: 「URLを開く」
  - URL: runkeeper://

アクション⑤: 「通知を表示」
  - タイトル: 🛑 Runkeeper
  - 本文: ランニングを停止してください

アクション⑥: 「テキスト」
  - 内容: stopped

アクション⑦: 「ファイルに保存」
  - 場所: iCloud Drive
  - ファイルパス: Shortcuts/runkeeper_state.txt
  - 上書き: オン
```

**「それ以外」（停止中 → 開始）:**

```
アクション⑧: 「URLを開く」
  - URL: runkeeper://

アクション⑨: 「通知を表示」
  - タイトル: 🏃 Runkeeper
  - 本文: ランニングを開始してください（スタートボタンをタップ）

アクション⑩: 「テキスト」
  - 内容: running

アクション⑪: 「ファイルに保存」
  - 場所: iCloud Drive
  - ファイルパス: Shortcuts/runkeeper_state.txt
  - 上書き: オン
```

**「if 終了」**

---

### 完成したショートカットの構成イメージ

```
[ファイルを取得] ← Shortcuts/runkeeper_state.txt
[変数を設定] currentState ← 上記の結果
[if] currentState = "running"
  ├─ [URLを開く] runkeeper://
  ├─ [通知を表示] "🛑 ランニングを停止してください"
  ├─ [テキスト] stopped
  └─ [ファイルに保存] → runkeeper_state.txt
[それ以外]
  ├─ [URLを開く] runkeeper://
  ├─ [通知を表示] "🏃 スタートボタンをタップしてください"
  ├─ [テキスト] running
  └─ [ファイルに保存] → runkeeper_state.txt
[if 終了]
```

## 背面タップに割り当てる

1. **設定** アプリを開く
2. **アクセシビリティ** → **タッチ** → **背面タップ**
3. **ダブルタップ**（または「トリプルタップ」お好みで）を選択
4. 一番下の **「ショートカット」** セクションから **「Runkeeper トグル」** を選択

---

## 使い方

| 状況 | 操作 | 結果 |
|------|------|------|
| ランニング開始したい | iPhoneの背面を2回タップ | Runkeeperが開く → 「スタート」ボタンをタップ |
| ランニング停止したい | もう一度背面を2回タップ | Runkeeperが開く → 停止操作が必要な場合はタップ |

---

## 注意事項

### Runkeeperのスタートボタンは手動タップが必要

iOSのセキュリティ制限により、**他のアプリのボタンを自動でタップすることはできません**。
このショートカットは：

- Runkeeperを素早く開く
- 現在の状態（開始/停止）を管理する

という役割を担います。Runkeeperが開いたら「スタート」または「停止」ボタンを1回タップするだけで済みます。

### 状態がズレた場合のリセット方法

iCloud Drive → Shortcuts → `runkeeper_state.txt` を削除すると、次回実行時に「停止中」状態から始まります。

---

## トラブルシューティング

| 問題 | 対処法 |
|------|--------|
| ショートカットがRunkeeperを開かない | `runkeeper://` のURLスキームが正しいか確認。Runkeeperを再インストール |
| 状態が正しく切り替わらない | iCloud Driveの同期が完了しているか確認 |
| 背面タップが反応しない | 設定 → アクセシビリティ → タッチ → 背面タップ で設定を再確認 |
| ケースをつけていると反応しない | 背面タップの感度は薄めのケースなら問題なし。厚いケースは反応しにくい場合あり |
